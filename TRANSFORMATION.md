# AI-PM-Designer-Pro 台灣蝦皮版改造規格書

**版本：** v1.0
**日期：** 2026-05-07
**目標：** 將 AI-PM-Designer-Pro 改造為專門的「台灣蝦皮商品圖片生成工具」

---

## 一、改造背景

### 1.1 現有系統

AI-PM-Designer-Pro 是一個基於 Google Gemini 的 AI 視覺行銷工具，採用 4 Phase 工作流：
- Phase 1：視覺策略制定（3 條行銷路線）
- Phase 2：內容企劃（8 張社群圖）
- Phase 3：市場分析（Google Search Grounding）
- Phase 4：內容策略（AI Studio / Gamma 提示詞）

### 1.2 目標系統

改造為使用 OpenAI image-2 API（通過中傳代理）的台灣蝦皮商品 Listing 圖片生成工具，專注於：
- 6 張 1:1 主圖
- 4-6 張 2:3 詳情圖
- SKU 圖（如有需要）
- 5 個 SEO 標題
- 產品描述 + CTA

### 1.3 參考規範

本改造主要參考以下規範：
- `taiwan-shopee-skincare-listing` SKILL.md
- `AGENTS.md`（中傳代理 API 使用指南）
- Shopee 台灣市場電商規範

## 三、API 對接改造

### 3.1 新增 API 服務模組

**檔案位置：** `services/imageGenService.ts`（新建）

**API 端點：**
- 文字生成：`POST https://api.xi-ai.cn/v1/chat/completions`
- 圖片生成：`POST https://api.xi-ai.cn/v1/images/generations`

**圖片模型：** `gpt-image-2`

**可用尺寸：**
| 尺寸 | 用途 |
|------|------|
| `1024x1024` | 主圖（1:1）、SKU 圖 |
| `1024x1536` | 詳情圖（2:3）|

**參數規範：**
```typescript
interface ImageGenerationParams {
  model: 'gpt-image-2';       // 固定
  prompt: string;               // 最大 32000 字符
  n?: number;                   // 默認 1
  size?: '1024x1024' | '1024x1536';
  quality?: 'low' | 'medium' | 'high';
  image?: string;               // base64 Data URI 格式
  background?: 'auto' | 'transparent' | 'opaque';
  output_format?: 'png' | 'jpeg';
}
```

**返回格式差異：**
- 無參考圖：返回 `{ data: [{ url: "..." }] }`
- 有參考圖：返回 `{ data: [{ b64_json: "..." }] }`

**超時設置：** `timeout >= 180 秒`

### 3.2 移除 Gemini 相關程式碼

**刪除檔案：**
- `services/geminiClient.ts`
- `services/geminiService.ts`

**刪除相依：**
- `components/` 中的 Gemini API Key Modal（替換為 OpenAI 格式）
- `utils/constants.ts` 中的 Gemini API 配置

### 3.3 API Key 處理

**儲存格式：** localStorage（不變）

**Key 名稱：** `openai_api_key`（新增），移除 `gemini_api_key`

**說明：** 用戶可自行設定 API Key，無需权限管控（本專案不共享）。

## 五、Prompt 規範

### 5.0 ⚠️ 原圖文字絕對不能修改（強制規則）

**【最高優先級】用戶上傳的圖片中的所有文字，無論是否為高風險，絕對不能做任何修改。**

#### 強制性要求：

1. **文字完全保真**
   - 產品圖片中的**每一個文字**都必須完整保留
   - 包括：高風險詞、中風險詞、低風險詞、品牌名、容量、成分表等
   - **不得**翻譯、繁簡轉換、刪除、遮蔽、模糊、重新繪製任何文字

2. **只能改變的範圍**
   - 產品的外觀變化（調整角度、調整光影、調整背景）
   - 場景改變（從浴室改到梳妝台、從室內改到室外）
   - 排版改變（改變佈局、調整大小比例）
   - **絕對不能重新繪製或修改圖片中的任何文字**

3. **強制寫入 Prompt 的指令**
   所有主圖、詳情圖、SKU 圖的 Prompt 都**必須包含**以下聲明：

   ```
   【最高優先級】原圖產品中的所有文字必須完全保真保留：
   - 不得翻譯、繁簡轉換、刪除、遮蔽、模糊或重新繪製任何文字
   - 品牌名、產品名、容量、成分表、說明文字等全部保持原樣
   - 只能改變產品的：外觀變化、場景背景、整體排版
   - 如果原圖使用簡體中文，保持簡體中文；如果是繁體中文，保持繁體中文
   ```

4. **模糊處理的對象限定**
   - **嚴禁**對原圖中的任何文字進行模糊/馬賽克處理
   - 馬賽克/模糊**只允許**用於用戶**手動選擇**的高風險文字區域
   - 必須由用戶在模糊工具中**親自框選**需要模糊的區域
   - AI **不得自行**決定模糊任何文字

5. **原圖使用簡體中文時**
   - 在 Prompt 中明確指定：「原圖產品使用簡體中文，保持不變」
   - **不得**將簡體中文轉換為繁體中文

#### 違反此規則的嚴重後果：
- 生成結果不符合以上要求時，**必須重新生成**
- 不得聲稱「已優化」或「已美化」而擅自修改文字
- 所有宣稱修改過的文字的行為都視為違規

### 5.1 主圖 Prompt 序列（6 張）

每張主圖 Prompt 必須包含：

```
1. 目標資產聲明：「目標資產：主圖 N / main-0N。只生成這一張獨立圖片，不要拼接其他圖片，不要生成總覽大圖。」
2. 嚴格尺寸要求：「這張圖必須是 1:1 正方形比例（square 1:1 aspect ratio）。」
3. 原圖文字保真聲明：「【最高優先級】原圖產品中的所有文字必須完全保真保留，不得做任何修改。」
4. 視覺風格聲明：「視覺風格大膽，高飽和，撞色，排版像潮流美妝社群貼文。Gen Z 彩色光影與俐落排版，不是傳統詳情頁或老式電視購物風格。」
5. 產品參考聲明：「產品參考：使用已驗證的產品底圖或用戶處理過的圖片，保持產品文字、包裝顏色與產品本體一致。」
6. 內容與文案排版
```

**主圖內容規範：**

| 序號 | 標題 | 內容方向 |
|------|------|----------|
| 1 | 商品第一眼封面圖 | 品牌+產品名清晰，一個安全短句副標，3 個特色標籤 |
| 2 | 商品亮點圖 A｜特色總覽 | 3 個安全賣點，概覽方向 |
| 3 | 使用情境圖 A｜人物/身體部位 | 展示人物使用產品的真實場景 |
| 4 | 使用情境圖 B｜生活場景/第二角度 | 不同角度或日常情境 |
| 5 | 商品亮點圖 B｜質地膚感 | 質地、使用感受方向 |
| 6 | 商品亮點圖 C｜成分概念/安心日常 | 成分概念、日常適配、信任感方向 |

### 5.3 Taiwan Gen Z 視覺要求

所有生成的 Prompt 都必須包含以下視覺要求：

```
視覺風格大膽，高飽和，撞色，排版像潮流美妝社群貼文。Gen Z 彩色光影與俐落排版，不是傳統詳情頁或老式電視購物風格。
```

**具體風格描述：**
- 色彩：高飽和、撞色、從包裝主色延伸，控制性的對比色
- 排版：潮流美妝社群貼文風格，俐落、不擁擠，非平衡但有秩序
- 光影：Gen Z 彩色光影、漸層光效、彩色陰影
- 字體：短標題、粗體關鍵詞、大標籤，1 個主要信息
- 避免：幼稚元素、迷因風格雜亂、大量折扣圖形、過度霓虹、emoji 牆

**Taiwan Gen Z 友好場景（選擇 2-3 個）：**
- 宿舍梳妝台（dorm room vanity）
- 通勤包（commute bag）
- 健身房/淋浴間架（gym/shower shelf）
- 辦公桌（office desk）
- 週末咖啡廳（weekend cafe table）
- 小公寓浴室（small apartment bathroom）
- 妝前準備（makeup-before-going-out）
- 共享房間備品（shared-room backup stock）
-  casual 送禮場景（casual gifting）

**禁止強制的道具元素：**
- 草药、人参、叶子、水滴
- 浴室場景（除非產品本身是沐浴相關）
- 粉色背景、金色背景
- 除非產品或用戶需求明確支持，否則不要強制使用這些元素

### 5.4 文案短語庫引用

標題和文案應從 `taiwan-genz-phrase-bank.md` 中選擇合適的短語組合，不要直接複製貼上短語列表，要使其看起來像正常的台灣電商文案。

**可用短語示例：**
- 日常好上手、妝前也好搭、小資補貨、宿舍備品
- 通勤包友善、早八出門也順手、一組配好不難選
- 自用送禮都自然、洗後不緊繃、舒服香感

### 5.5 SEO 標題規範

```
- 5 個標題，每個至少 50 個中文字（不含 emoji、空格、標點）
- 使用空格分隔關鍵詞，不要用 | 或 ｜
- 自然分佈 3-4 個 emoji，不要全部放在開頭或結尾
- 5 個標題切入點要不同：產品類型、使用場景、SKU/送禮、質地感受、買家意圖
- 可用台灣 Gen Z 電商語感詞：日常好上手、妝前也好搭、小資補貨、宿舍備品、通勤包友善、早八出門也順手
```

**SEO 標題字符計算：** 中文 50 個字符（不含 emoji、空格、標點符號）

### 5.6 產品描述規範

```
- 語氣像親切大姐姐陪買家判斷，自然、貼近、實用
- 2-4 個手機友好段落
- 避免生硬用詞：這款產品、本產品、該產品、此商品、本商品、產品定位為
- 開頭可用：如果你也喜歡...、日常保養想找...、我會把它放在...、很適合想要...的人
- 全篇使用 4-8 個自然 emoji，放在短語或段落轉換後
- 結尾包含自然 CTA
```

### 5.7 詳情圖第 3 張的條件替換

**預設：** 詳情圖 3 為「適合誰/怎麼選圖」

**條件替換為「套組使用順序/建議搭配方式圖」：**
- 套組、多產品流程
- 工具/儀器
- 步驟不直覺的面膜類產品
- 用戶明確要求時

### 5.8 SKU 圖統一視覺系統

當需要生成 SKU 圖時，所有 SKU 圖必須使用**統一的視覺系統**：

| 項目 | 統一規範 |
|------|----------|
| **版式** | 所有 SKU 圖使用相同的排版結構（如：上方標題、中間產品、下方內容） |
| **色彩風格** | 沿用產品包裝主色，所有 SKU 圖使用同一色系，只用小面積輔助色區分 |
| **字體層級** | 主標題 / 副標題 / 標籤 / 底部小字 固定層級 |
| **產品位置** | 產品比例、角度、陰影、擺放位置保持一致 |
| **標籤樣式** | 圓角標籤 / 小 icon / SKU 內容卡片樣式保持一致 |
| **可變元素** | 只允許 SKU 名稱、數量、內容、場景標籤、小道具與小面積輔助色變化 |

**不允許的行為：**
- 每個 SKU 圖設計成不同活動風格
- 不同模板、不同品牌風格
- 重新設計產品比例或角度

### 5.9 圖片檔案命名規則

### 6.2 圖片文字識別流程

```
1. 用戶上傳產品圖片
2. 使用 GPT-5.5 Vision 識別圖中所有可見文字
3. AI 評估每個文字的風險等級：
   - high（高風險）：功效宣稱詞
   - medium（中風險）：可能引發平台審核關注
   - low（低風險）：品牌名、容量規格等
4. 用戶確認要模糊的高風險詞
5. 生成合規產品底圖（僅對高風險詞做局部馬賽克/模糊）
6. 後續所有主圖、詳情圖、SKU 圖都使用這張已驗證的底圖
```

### 6.3 產品底圖工作流（強制性要求）

**當原圖含高風險/違規詞或需要保持產品文字一致性時，必須遵守以下流程：**

#### 強制性規則：

1. **必須先生成並驗證產品底圖**
   - 先為每個產品/SKU 變體/包裝版本生成一張 `product-base-{序號}` 合規底圖
   - 在生成任何主圖、詳情圖、SKU 圖之前，必須先驗證這張底圖

2. **後續所有圖片必須引用已驗證的產品底圖**
   - 所有主圖 Prompt 都必須引用：`產品參考：使用已驗證的 product-base-{序號} 合規產品底圖`
   - 所有詳情圖 Prompt 都必須引用相同的底圖
   - 所有 SKU 圖 Prompt 都必須引用對應的底圖

3. **禁止重複遮蔽**
   - 後續圖片**不能**對原圖做不同位置、不同強度或不同範圍的遮蔽
   - 如果底圖驗證不通過（文字轉換錯誤、過度模糊、改變包裝、顏色不符、產品不符），必須**重新生成底圖**，而不是在後續圖中調整遮蔽

4. **遮蔽精準度要求**
   - 只遮蔽高風險詞字符，用緊密邊距覆蓋
   - 保持相鄰的安全文字清晰：品牌、品名、成分、容量、條碼等
   - 產品形狀、logo 區域、包裝顏色、瓶子、蓋子、盒子、紋理、陰影等細節**完全不變**

5. **原圖文字保真度**
   - 如果原圖包裝使用簡體中文，在產品底圖中**保持簡體中文**
   - 台灣正體中文只應用於**新添加**的文案、標題、CTA、疊加層

#### 12.4.2 風險等級標註
| 等級 | 標註顏色 | 處理方式 |
|------|----------|----------|
| **高風險** | 🔴 紅色 | 默認選中，自動模糊 |
| **中風險** | 🟡 黃色 | 可選是否模糊 |
| **低風險** | 🟢 綠色 | 不處理 |

#### 12.4.3 模糊預覽
- 在原圖上標註將被模糊的區域（紅色邊框）
- 可調整模糊強度（輕度/中度/重度）
- 支援局部放大查看模糊效果

#### 12.4.4 ⚠️ 重要：關於原圖文字的處理原則

**【最高優先級】用戶上傳的圖片中的所有文字，無論是否為高風險，絕對不能做任何修改。**

| 允許的操作 | 禁止的操作 |
|------------|------------|
| 調整產品外觀（角度、光影、背景） | 修改、刪除、遮蔽任何文字 |
| 改變場景背景 | 翻譯、繁簡轉換任何文字 |
| 調整整體排版 | 對任何文字進行模糊/馬賽克處理 |
| 變化產品呈現方式 | 重新繪製任何文字 |

**只有當用戶需要對圖片中的高風險文字進行處理時，才可以：**
1. 由用戶**親自**在模糊工具中框選需要處理的區域
2. 對框選的區域進行**局部模糊**處理
3. 處理後的圖片作為新的參考圖使用

**模糊工具的使用流程：**


**模糊工具技術實現：**

```typescript
interface BlurTool {
  // Canvas 繪製
  canvas: HTMLCanvasElement;

  // 選取模式
  mode: 'select' | 'draw';

  // 已選取的模糊區域
  regions: BlurRegion[];

  // 添加模糊區域
  addRegion(x: number, y: number, width: number, height: number): void;

  // 刪除模糊區域
  removeRegion(index: number): void;

  // 應用模糊效果
  applyBlur(imageData: ImageData, intensity: 'light' | 'medium' | 'heavy'): ImageData;

  // 導出處理後的圖片
  exportImage(): string; // base64 Data URI
}

interface BlurRegion {
  x: number;      // 0-1  normalized
  y: number;      // 0-1  normalized
  width: number;  // 0-1  normalized
  height: number; // 0-1  normalized
}
```

**模糊強度參數：**
| 等級 | 參數值 |
|------|--------|
| 輕度 | blur radius: 5px |
| 中度 | blur radius: 10px |
| 重度 | blur radius: 20px |

#### 12.4.5 處理後的圖片使用

1. 用戶完成模糊處理後，點擊「確認」
2. 系統生成一張新的圖片（僅模糊區域被處理）
3. 新圖片替換原圖作為後續生成的參考圖
4. 在 Prompt 中明確指定使用處理後的圖片

```typescript
  promptText: string;         // 當時使用的 Prompt
  parameters: {               // 當時使用的參數
    size: string;
    quality: string;
    model: string;
  };
  result: {
    success: boolean;
    imageUrl?: string;        // 成功時的圖片 URL
    error?: string;           // 失敗時的錯誤訊息
  };
  createdAt: number;          // 生成時間
  duration: number;           // 生成耗時（秒）
}
```
